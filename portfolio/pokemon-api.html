De opdracht

Voor het vak API van de minor Web Design & Development was de opdracht om een website te maken die gebruik maakt van een API. Ik heb een Pokémon website gemaakt met behulp van PokéAPI. Je moest minimaal een overzichtspagina en een detailpagina hebben en gebruik maken van minstens 2 web API's.

De uitvoering

Ten eerste heb ik een overzichtspagina gemaakt waar de Pokémon op geladen worden. Deze worden allemaal uit de API opgehaald. Ik heb ze de styling gegeven van Pokémon kaarten. Daarnaast heeft elke Pokémon een eigen detailpagina waar allerlei statistieken te zien zijn. Omdat ik gebruik heb gemaakt van Astro, kon ik met 1 component een pagina maken voor alle Pokémon, waardoor ik niet honderden verschillende paginas hoef te schrijven.

Daarnaast heb ik ook een pagina gemaakt met een canvas een wereld waarin je rond kunt lopen en Pokémon kunt vangen. Deze pokemon worden willekeurig uit het API geladen en elk stuk gras heeft andere type Pokemon. Daarbij heeft elke Pokemon een eigen geluid en catch rate die ook uit de API komen. Deze pokemon worden lokaal opgeslagen en kun je zien tijdens de encounter.

Verder heb ik mezelf goed uit kunnen dagen met wat ik heb kunnen doen en ben trots dat ik het spel zo goed na kon maken. Het enige jammere is dat ik niet de tijd had om een battle systeem toe te voegen, omdat ik mijn laatste tijd wilde spenderen aan het polishen van animaties, styling en audio. Ik vind het wel bizar dat dit allemaal met alleen JavaScript kan. Ik had voor dit project Astro gebruikt, wat inhield dat ik componenten kon maken, zodat ik niet meerdere keren HTML hoefde te schrijven voor bv. de knoppen en alle detailpagina's.
